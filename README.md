# Реализация

### 1. Модулятор 

`QAM_gray`:

- Принимает вектор битов и тип QAM (4, 16, 64)
- группирует биты в I и Q компоненты по 2 (для QPSK), по 4 (для 16‑QAM) или по 6 (для 64‑QAM)
- Преобразует двоичные числа I и Q в код Грея:  `gray = value ^ (value >> 1)`
- Вычисляет уровни(амплитуду)

**Код Грея** — это способ представления чисел, где соседние значения отличаются только в одном бите.

В QAM созвездии ближайшие друг к другу символы отличаются малым сдвигом по фазе/амплитуде – именно такие ошибки наиболее вероятны при шуме. Если эти ближайшие символы закодировать с помощью кода Грея, то при ошибочном приёме соседнего символа исказится только один бит, а не несколько. Это сильно снижает BER (Bit Error Rate) при том же SNR, потому что меньше битов портится на одну ошибку символа.
  
  ### 2. Канал AWGN

Функция AWGN добавляет к каждому символу случайный комплексный гауссовский шум с нулевым средним и заданной дисперсией σ². Чем больше дисперсия, тем сильнее размазаны точки на созвездии. Реализована через std::normal_distribution<float> и статический генератор std::mt19937.


  ### 3. Демодулятор

  Функция QAM_demapper выполняет обратные преобразования:

- Денормировка символа 
- Округление до ближайшего целого уровня
- Преобразование уровня в код Грея
- Обратное преобразование Грея → двоичное число по таблице gray2bin[ k-1 ][ gI ]
- Извлечение битов
    
  ### 4. Расчёт BER и сохранение данных

Программа автоматически прогоняет один и тот же набор битов через 21 уровень шума – от очень слабого (дисперсия 0.01) до заметного (дисперсия 1.0). Для каждого уровня демодулируются символы, сравниваются полученные биты с исходными и считается доля ошибок (BER).
Все результаты сохраняются в файл ber_results.csv. Запускается графический интерфейс с передачей всех рассчитанных данных
